概要
どんな作品
DirectX 12 環境で VAT（Vertex Animation Texture）による頂点変形を実装し、 マイク音声をリアルタイム解析することで波形アニメーションを制御する映像表現の検証作品。 音量が振幅に、ピッチが波の速度に対応し、発声中のみディザリングとHSVレインボーが発動する。
なぜ作った？
流体シミュレーションをゲームに活用したいという動機から出発。 負荷の高い流体計算の代替として VAT ＋ ディザリングで似た映像効果が得られると仮説を立て、 さらに JunkShooting で培った音声経験を組み合わせることで 「音で動く流体表現」というプロシージャルな映像体験を目指した。

① リアルタイム音声解析 ─ VAT_demo.cpp（VATAudio）
WinMM コールバックで RMS（音量）とゼロクロス数（ピッチ推定）を計算し std::atomic に書き込む。 描画ループはロックなしで最新値を読む。Update() で線形補間（smooth=0.15）してガクガクを防ぐ。 
// コールバック内：バッファが埋まるたびに呼ばれる
static void CALLBACK WaveInProc(...) {
    const int16_t* samples = (const int16_t*)hdr->lpData;
    // RMS（音量）
    double acc = 0.0;
    for (int i=0; i<count; ++i) acc += (double)samples[i] * samples[i];
    s_lastRms.store( (float)std::sqrt(acc/count) / 32768.0f );
    // ゼロクロス（ピッチ推定）
    hz = (float)crossings * kSampleRate / (2.0f * count);
    s_lastHz.store(hz);
    waveInAddBuffer(hwi, hdr, sizeof(WAVEHDR)); // 再投入
}
// 毎フレーム：スムージングして描画側に渡す
static void Update() {
    const float smooth = 0.15f;
    s_amp.store(  Lerp(s_amp.load(),   RmsToAmp(s_lastRms.load()), smooth) );
    s_speed.store(Lerp(s_speed.load(), HzToSpeed(s_lastHz.load()), smooth) );
}
